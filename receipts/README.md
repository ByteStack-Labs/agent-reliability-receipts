# Receipts

Each receipt is the output of pointing the [agent-reliability](https://github.com/ByteStack-Labs/claude-plugins)
plugin at a fixture, a small program built on purpose to fail in one real way, in
this repo and letting it work. A receipt is not written by hand. It is generated by
the tool and committed exactly as emitted, including any stylistic glyphs, so it is
a real record of the skill running, not a description of what it might do. It is
deliberately not hand-edited to match house style; the unedited artifact is the point.

## Generate the invoice-extraction receipt

From a Claude Code session in this repository, with the plugin installed:

```
/agent-reliability:production-autopsy
```

Point it at `fixtures/invoice-extraction`. The skill frames the gap between the
controlled test the model passes and the real-world input it fails on, quantifies
it by slice, the group of cases sharing one trait, from the saved `shift_type`,
isolates the root cause (the model keyed on field position rather than meaning),
and writes a verification script plus a report. Commit whatever it produces under
`receipts/invoice-extraction/`.

The verification script it writes is standard-library only and reproduces every
number in its report, so a reader can clone this repo and confirm the finding
without a model or a GPU:

```
python3 receipts/invoice-extraction/verify.py
```

It exits non-zero if any figure fails to reproduce.
